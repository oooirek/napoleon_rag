# ТОЧНОСТЬ

как при больших файлах не терять точность овтета
    - поменять эмбеддинг модель
    - Rerank после retrieval: взять top-k из Qdrant и прогнать reranker (cross-encoder). Это часто дает +30–50% качества
    - Гибридный поиск: сочетание векторного (semantic) и BM25 (keyword). Это сильно повышает точность для “точных” вопросов. В Qdrant можно сделать hybrid через
    отдельный BM25 индекс или через внешнюю библиотеку + rerank
    - Пересмотреть чанкование: уменьшить chunk_size, увеличить “умный” overlap, использовать разные стратегии для текста и таблиц. На длинных PDF чанк по 700 может смешивать темы.
    - Добавить метаданные и фильтры: doc_id, page, section, source, title. Потом можно фильтровать/взвешивать по документу или разделу



# СПОСОБЫ АВТОМАТИЗАЦИИ ТЕСТИРОВАНИЯ

Ниже практичные способы автоматизировать проверку качества ответов в RAG, от простого к продвинутому.

1) Набор эталонных вопросов (обязательный минимум)

Сделай таблицу question → expected answer → source doc/page.
30–100 вопросов, покрывающих ключевые сценарии.
Это твой “регрессионный тест” качества.
2) Автоматическая проверка соответствия

Для “фактов” сравнивай ключевые сущности: числа, даты, коды, названия.
Простые метрики: exact match, contains, regex по числам/датам.
Для длинных ответов — сравнение “по смыслу” через similarity (embedding‑similarity).
3) Проверка retrieval (до LLM)

Отдельно проверь: находится ли правильный чанк в top‑k.
Метрики: Recall@k, Hit@k.
Это помогает понять, проблема в поиске или в генерации.
4) Отдельная проверка таблиц

Делай вопросы, которые точно лежат в таблицах.
Сравнивай извлечённые значения напрямую (числа/строки).
Если retrieval не нашёл нужную строку — проблема в индексировании таблицы.
5) Автоматическая “LLM‑проверка”

Второй LLM оценивает, насколько ответ соответствует вопросу и контексту.
Хорошо для раннего выявления галлюцинаций, но не единственный критерий.
Лучше использовать как вспомогательный сигнал.
6) Версионность индекса

Привязывай результаты тестов к версии индекса (например, collection_name_v3).
Так ты видишь прогресс/регресс после изменений чанкера/эмбеддингов.
